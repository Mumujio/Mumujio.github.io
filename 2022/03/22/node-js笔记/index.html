fs文件系统模块

  1. fs.readFile() 方法：可以读取指定文件中的内容

    fs.readFile(path[,options],callback)

    • 参数1：必选参数，字符串，表示文件标识
    • 参数2：可选参数，表示以什么编码格式来读取文件
    • 参数3：必选参数，文件读取完毕完，通过回调函数拿到读取的结果

  1. fs.writeFile() 方法：想指定的文件中写入内容

    fs.writeFile(file,date[,options],callback)

    • 参数1：必选参数，需要指定一个文件路径的字符串，表示文件的存放路径
    • 参数2：必选参数，表示要写入的内容
    • 参数3：可选参数，表示以什么格式写入文件内容，默认值是utf8
    • 参数4：必选参数，文件写入完成后的回调函数

path模块

  • path模块是Node.js 官方提供的，用来处理路径的模块
    1. path.join() 方法：用来将多个路径片段拼接成一个完整的路径字符串
    • 凡是涉及路径拼接的操作都要使用path.join()进行处理
    1. path.basename() 方法：用来从路径字符串中，将文件名解析出来
    • path.basename(path[,ext]):
      • path 必选参数，表示一个路径的字符串
      • ext 可选参数，表示文件拓展名
      • 返回： 表示路径中的最后一部分
    1. path.extname() 方法：可以获取路径中的拓展名部分

http 模块

  • http 模块用来创建 web 服务器的模块

  • 在node.js中，我们不需要使用第三方服务器软件，因为我们可以基于Node.js 提供的http 模块，通过代码手写服务器软件，从而对外提供web服务

  • IP地址：互联网上每台计算机的唯一地址

    • 互联网中每台Web服务器，都有自己的IP地址
    • 在开发期间，自己的电脑是一台服务器，也是一个客户端（127.0.0.1)

    • 域名与IP地址是一一对应的关系 (127.0.0.1 === localhost）
    • 域名服务器就是提供IP地址和域名之间转换服务的服务器

模块化

  • 遵守固定的规则，把一个大文件拆成独立并互相依赖的多个小模块

    • 好处：
      1. 提高了代码的复用性
      2. 提高了代码的可维护性
      3. 可以实现按需加载

  • Node.js 将模块分为了3大类：

    1. 内置模块：有官方提供，如 fs、path、http等
    2. 自定义模块：用户创建的每个 .js 文件，都是自定义模块
    3. 第三方模块：由第三方开发出来的模块，使用前需要先下载

  • require()方法：可以加载需要的各种模块进行使用

  • 模块在引用时会被执行一次

  • Node.js 的模块作用域：与函数作用域类似，在自定义模块中定义的变量和方法等成员，只能在当前模块内访问。这种访问限制，叫做模块作用域

    • 好处：防止全局变量污染的问题

  • module对象：在每个 .js 自定义模块中都一个 module对象，它里面存储了和当前模块有关的信息

    • module.exports对象：在自定义模块中可以使用该对象，将模块内的成员共享出去，供外界使用。**外界使用 require() 方法导入自定义模块时，得到的就是module.exports 所指向的对象
      • 默认情况下，module.exports = {}
    • 使用 require()方法导入模块时，导入的结果，永远以 module.exports 指向的对象为准
    • exports 对象：默认情况下，exports 和 module.exports 指向同一个对象，最后永远以 module.exports 指向的对象为准

  • Node.js 中的模块化规范

    • CommonJS规定：
      1. 每个模块内部，module变量代表当前模块
      2. module变量是一个对象，他的exports属性（即 module.exports）是对外的接口
      3. 加载某个模块时，其实是加载该模块的 module.exports属性。require() 方法用于加载模块

  • 模块的加载机制：

    1. 优先从缓存中加载
      1. 模块在第一次加载后会被缓存。都会优先从缓存中加载，从而提高模块的加载效率
    2. 内置模块的加载机制
      1. 内置模块的加载优先级最高
    3. 自定义模块的加载机制
      1. 使用require()加载自定义模块时，必须指定以./ 或 ../开头的路径标识符。如果没有，node会把它当作内置模块和第三方模块进行加载
      2. require() 导入自定义模块时省略文件的拓展名：
        • 确切的文件名 -> .js -> .json -> .node -> 加载失败
    4. 第三方模块的加载机制
    5. 以目录作为模块
